# Spend Receipts — AI Agent Budget Audit

A hackathon demo of an ERC-4337 smart account that holds USDC and enforces
per-category caps on every payment an AI agent makes. Over-cap payments
revert at the protocol. Every accepted payment becomes a refundable
on-chain receipt that links to Basescan.

The brief this implements is `demo-brief.md` in the repo root.

```
  ERC-4337 smart account
  ──────────────────────
  owner:    Marcus         agent:    agent script
       │                          │
       │  setCaps / fund          │  execute(vendor, amt, cat)
       │  refund                  │     │
       └────────┬─────────────────┘     │
                ▼                       ▼
       ┌───────────────────────────────────────┐
       │         SpendAccount contract         │
       │                                       │
       │  cap[Research] = 10 USDC / week       │
       │  cap[Images]   =  5 USDC / week       │
       │  cap[Other]    =  0 USDC / week       │
       │                                       │
       │  if (spent + amt > cap) REVERT        │
       │  else transfer USDC + emit receipt    │
       └───────────────────────────────────────┘
                │                       │
                ▼                       ▼
        MockVendor (Research)    MockVendor (Images)    MockVendor (Other)
```

## Quickstart

```sh
make demo
```

That is the entire quickstart. `make demo` builds and tests the contracts,
starts a local Anvil node, deploys everything, syncs the deployment
addresses into the frontend, and starts Next.js on http://localhost:3000.

Prerequisites (already installed in the demo environment):

- Node 20+
- Foundry (`curl -L https://foundry.paradigm.xyz | bash && foundryup`)
- A MetaMask browser extension pointed at `http://localhost:8545` with
  chainId `31337`

After `make demo` prints the ready banner:

1. Open http://localhost:3000
2. On the **Setup** tab, click `CONNECT METAMASK` and approve
3. Click `SEED DEMO STATE` to mint 20 USDC into the smart account and
   apply the brief's caps in one transaction
4. Switch to the **Activity** tab and click the three payment buttons:
   `Research API Co / $3`, `Stock Image Co / $4`, `Contractor / $8`
5. Watch the third one revert with `CapExceeded(Other, 0, 0, 8000000)`
6. Click `REFUND` on the $4 receipt
7. Open the **Summary** tab to see Spent $3.00 / Refunded $4.00 /
   Blocked $8.00 plus per-row Basescan links

## What the demo proves

The smart account contract is wired so that:

| Brief criterion                                                    | Verified by                                                  |
| ------------------------------------------------------------------ | ------------------------------------------------------------ |
| Over-cap payment reverts at contract level                         | `test_AgentPayOverCap_RevertsAndEmitsBlocked`                |
| Each accepted payment appears within ~2s on Live Activity           | `useReadContract` polls every 4s + `useWatchContractEvent`    |
| Refund returns USDC within 10s; row updates without page reload    | Same hooks; `spentThisWeek` decrements on refund             |
| Caps persist per-category; refund restores cap space               | `test_RefundReturnsUsdcAndRestoresCap`                       |
| Blocked attempt remains visible (failed tx on Basescan)            | `PaymentBlocked` event captured by dashboard                 |

`forge test` exercises all six demo steps in `test_DemoFlow_SixSteps`
plus 15 supporting tests. The cast walk-through in the commit message
verified the same flow end-to-end against a fresh Anvil instance.

## Demo-only shortcuts (read before judging)

- **USDC is a local mock.** `MockUSDC.mint()` is permissionless. On a
  real deployment you swap the immutable `usdc` to canonical USDC
  (`0x036CbD53842c5426634e7929541eC2318f3dCF7e` on Base Sepolia) and
  remove `demoSeed()`. The `setCaps` / `fund` / `execute` / `refund`
  flow is unchanged.
- **Vendors auto-approve the smart account** at deploy time so refunds
  work without manual approval. In production each vendor would approve
  on receiving payment.
- **Single EOA plays both owner and agent** in the demo. In production
  they are distinct keys — the agent is a hot key, the owner is the
  cold key. The contract enforces this distinction via the `onlyAgent`
  and `onlyOwner` modifiers but the demo collapses them for reviewer
  ergonomics.
- **Local Anvil instead of Base Sepolia.** Base Sepolia works exactly
  the same — point `ANVIL_RPC` at `https://sepolia.base.org` and
  re-run `make deploy`. The brief itself says testnet is fine.
- **No real ERC-4337 bundler.** `execute()` is called directly by the
  agent EOA. For production, swap the SpendAccount into a SimpleAccount
  pattern behind a bundler (StackUp, Pimlico, Base). The cap logic
  stays exactly as is.

## How to present (5 min)

1. **0:00** Open Setup. Show the wallet card disconnected. Connect
   MetaMask — the panel flips to CONNECTED with the owner address.
2. **0:30** Click `SEED DEMO STATE`. One transaction: 20 USDC appears
   in the smart account, caps set to 10/5/0.
3. **1:30** Move to Activity. Point out the three payment buttons —
   the agent's x402 calls.
4. **2:00** Click `Research API Co / $3`. The row appears in the feed
   in <2s with vendor, amount, category, status `ACCEPTED`.
5. **2:30** Click `Stock Image Co / $4`. Same, totals update.
6. **3:00** Click `Contractor / $8`. The button flips to
   `BLOCKED / CAP-EXCEEDED`. A red BLOCKED row appears with a Basescan
   link to the failed tx.
7. **3:30** Click `REFUND` on the $4 receipt. Row flips to REFUNDED,
   balance moves back, cap space restored.
8. **4:00** Switch to Summary. End-state totals: Spent $3 / Refunded $4
   / Blocked $8. Click any Basescan link to verify on a block explorer.
9. **4:30** Click `EXPORT CSV` to ship the audit trail.

If a reviewer wants to dig in:

- `cd contracts && forge test -vvv` — 16 tests, full demo flow
  reproduced in `test_DemoFlow_SixSteps`
- `cat contracts/deployments/latest.json` — the on-chain addresses the
  frontend is talking to
- `cast block-number --rpc-url http://127.0.0.1:8545` — confirms the
  chain is alive

## Stopping the demo

```sh
make demo-stop   # kills anvil, leaves .next build cache
make clean       # also wipes forge cache + broadcast + frontend build
```

## What is intentionally NOT in scope

Per the brief:

- No fiat on-ramp. USDC enters via the demo seed.
- No Stripe / ACH / PayPal integration.
- No mobile native. Responsive web only.
- No cross-session persistence beyond what the contract emits.
- No gas refund on rejected calls.
